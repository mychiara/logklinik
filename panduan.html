Panduan Instalasi E-Klinik

Migrasi dari Google Sheets ke Database Supabase (Full PWA)

1

Persiapan Proyek Supabase

Langkah pertama adalah membuat "rumah" baru untuk database Anda.

  • Masuk ke supabase.com dan Login/Daftar.
  • Klik New Project. Pilih nama (misal: E-Klinik) dan tentukan Password Database.
  • Tunggu hingga proses Setting Up Project selesai (biasanya 2-3 menit).
Pilih Region yang terdekat dengan pengguna (misal: Singapore) agar akses aplikasi lebih cepat.
2

Membuat Struktur Tabel (SQL)

Agar database siap menampung data Mahasiswa, Preseptor, dan Logbook, kita harus menjalankan skema SQL.

  • Di dashboard Supabase, klik menu SQL Editor (ikon terminal di kiri).
  • Klik New Query.
  • Buka file supabase-schema.sql dari folder proyek Anda, lalu Copy dan Paste semua isinya ke editor.
  • Klik tombol Run.
Jika muncul error "relation users already exists", berarti tabel sudah ada. Tambahkan perintah DROP TABLE IF EXISTS... di awal atau abaikan saja jika tabel sudah terbuat.
3

Matikan RLS (Row Level Security)

Supabase secara default mengunci akses tabel secara ketat. Kita perlu memberi izin akses agar aplikasi bisa login.

Jalankan kueri berikut di SQL Editor:

-- Matikan proteksi (RLS) sementara agar API bisa diakses publik ALTER TABLE users DISABLE ROW LEVEL SECURITY; ALTER TABLE presensi DISABLE ROW LEVEL SECURITY; ALTER TABLE logbook DISABLE ROW LEVEL SECURITY; ALTER TABLE kompetensi DISABLE ROW LEVEL SECURITY; ALTER TABLE tempat_praktik DISABLE ROW LEVEL SECURITY; ALTER TABLE prodi DISABLE ROW LEVEL SECURITY; ALTER TABLE jadwal DISABLE ROW LEVEL SECURITY; ALTER TABLE laporan DISABLE ROW LEVEL SECURITY; ALTER TABLE settings DISABLE ROW LEVEL SECURITY; ALTER TABLE kelompok DISABLE ROW LEVEL SECURITY;

Langkah ini memastikan error 406 Not Acceptable pada login tidak muncul lagi.

4

Konfigurasi API di Aplikasi

Hubungkan file proyek Anda dengan URL dan Kunci API dari Supabase.

  • Buka menu Project Settings (ikon gir) > Pilih API.
  • Salin Project URL dan anon public key.
  • Buka file js/supabase-config.js di folder proyek Anda.
  • Tempelkan kedua nilai tersebut ke dalam variabel yang tersedia:
const SUPABASE_URL = "https://xyz.supabase.co";
const SUPABASE_ANON_KEY = "eyJhbGc...k0M2Y";
5

Hosting & Anti-Pause (GitHub)

Karena Supabase Free Plan bisa mati otomatis jika tidak ada aktivitas selama 1 minggu, kita pasang Robot GitHub.

  • Upload proyek Anda ke GitHub.
  • Masuk ke tab Settings > Secrets > Actions di GitHub Anda.
  • Daftarkan Secret: SUPABASE_URL dan SUPABASE_ANON_KEY.
  • Robot otomatis akan berjalan setiap hari Senin berdasarkan file di folder .github/workflows/.

Instalasi Selesai!
