Panduan Instalasi & Konfigurasi Database Supabase (Production Ready)

1

Menyiapkan Project Supabase

Langkah awal untuk membangun backend yang stabil dan cepat.

  • Buat akun di supabase.com.
  • Buat New Project. Simpan Database Password Anda baik-baik.
  • Tungu hingga status project menjadi Active.
Pilih Region Singapore untuk performa terbaik jika Anda berada di Indonesia.
2

Membangun Skema Database

Eksekusi perintah SQL untuk membuat tabel-tabel utama termasuk sistem Penilaian Akhir yang baru.

  • Masuk ke menu SQL Editor di Sidebar kiri Supabase.
  • Klik New Query.
  • Copy seluruh isi file supabase-schema.sql dari folder project ini.
  • Tempelkan ke dalam editor dan klik Run.
Pastikan semua tabel berhasil dibuat (centang hijau). Jika ada tabel yang gagal, jalankan ulang kueri tersebut secara spesifik.
3

Keamanan (RLS Policy)

PENTING: Jangan mematikan RLS di server produksi. Gunakan kebijakan akses agar data tetap aman.

Jalankan kueri ini di SQL Editor untuk memberi izin akses:

-- Memberi izin Baca/Tulis untuk Aplikasi ALTER TABLE users ENABLE ROW LEVEL SECURITY; CREATE POLICY "Public Read" ON users FOR SELECT TO anon USING (true); CREATE POLICY "Public Write" ON users FOR INSERT TO anon WITH CHECK (true); -- Lakukan hal yang sama untuk tabel lain (presensi, logbook, laporan) -- Atau jalankan script RLS lengkap dari file audit_results.md

Kebijakan ini memungkinkan aplikasi web Anda berkomunikasi dengan database menggunakan anon key secara aman.

4

Menghubungkan API

Konfigurasi koneksi antara frontend (browser) dan backend (Supabase).

  • Buka Project Settings (gir bawah) > API.
  • Salin Project URL dan anon keys.
  • Buka file js/supabase-config.js di komputer Anda.
  • Update variabel berikut:
const SUPABASE_URL = "https://proyek-anda.supabase.co"; const SUPABASE_ANON_KEY = "eyJhbGc...pasted-from-dashboard";
